# MR レビュー対応スキル

## 役割

GitLab 上の Merge Request に寄せられたレビューコメントに対応し、修正を実装してコミットを作成するタスクを自動化します。

## トリガー

ユーザーが以下の形式でコマンドを実行したときに起動します：

```
/mr-review <MR 番号>
```

例：`/mr-review 4`

## ワークフロー

### 1. MR 詳細の取得

MR の詳細情報を取得します。

### 2. 未解決スレッドの抽出

取得した MR 情報から、`discussion` フィールドを解析し、未解決（`resolved: false`）のスレッドのみを抽出します。解決済みのスレッドはスキップします。

### 3. 優先順位の自動判定

各未解決スレッドを技術的重要度で自動判定し、優先順位付けします。

| 重要度 | 判定基準 | 例 |
|--------|----------|-----|
| **高** | バグ修正、セキュリティ、機能欠落、テスト不足 | 「テストが不足している」「バグが潜んでいる」 |
| **中** | リファクタリング、コード品質、設計改善 | 「enum class を使った方が良さそう」「可読性を向上」 |
| **低** | ドキュメント、スタイル、コメント追加 | 「コメントを追加してほしい」「ドキュメントを更新」 |

判定結果はユーザーに確認し、対応順序の承認を得ます。

### 4. 修正方針の設計

各コメントに対応する修正内容を設計します。関連する変更は 1 つのコミットにまとめます。

修正方針をユーザーに提示し、承認を得ます。

### 5. 修正の実装（逐次）

各コメントに対して、以下の手順で修正を実装します：

1. 該当ファイルの修正
2. テストの追加（正常系・異常系・境界値）
3. git diff で変更内容を確認
4. ユーザーに確認
5. テスト実行（`ctest --test-dir build -C Debug --output-on-failure`）
6. テスト結果を確認
7. コミット作成

### 6. コミットメッセージの形式

スキル `commit-message` のガイドラインに従ってください。

### 7. コミット作成

各コメントに対して 1 つのコミットを作成します（逐次コミット）。

コミットメッセージをユーザーに確認し、承認を得てから作成します。

詳細は [`commit-control`](../commit-control/SKILL.md) スキルを参照してください。

### 8. プッシュ指示

全コミット作成完了後、以下のコマンドをユーザーに提示します（実行は行いません）：

```bash
git push
```

## 判断基準

### 優先順位判定ルール

- **高重要度**: バグ修正、セキュリティ、機能欠落、テスト不足
- **中重要度**: リファクタリング、コード品質、設計改善
- **低重要度**: ドキュメント、スタイル、コメント追加

自動判定後、必ずユーザーに確認します。

### テスト追加ルール

以下のテストは自動的に追加します：

- **正常系**: 基本機能の動作確認
- **異常系**: 無効入力のエラーハンドリング
- **境界値**: 限界値のテスト

プロジェクト固有のテスト（例：文字種検証）は、レビューコメントで明記された場合のみ追加します。

### コミット分割ルール

- 各レビューコメントを別コミットにします
- 関連する変更は 1 つのコミットにまとめます
- 1 コミットには 1 つの論理的な変更のみを含めます

## エッジケース処理

### コメントが「対応済み」の場合

既に実装済みのコメントはスキップし、ユーザーに報告します。

### コメントが曖昧な場合

実装方針が明確でない場合は、ユーザーに確認を取ります。

### 実装不可能なコメント

技術的に不可能な場合は、理由を説明してユーザーに報告します。

## 注意事項

1. **プッシュは実行しない**: コミット作成までで止まり、プッシュはユーザーが手作業で実施します。

2. **確認のタイミング**:
   - 修正方針の実装前
   - 変更内容のコミット前
   - テスト結果の承認
   - コミットメッセージの承認
   - 優先順位の自動判定結果

3. **複数選択肢がある場合**: 実装中に複数の選択肢がある場合は、両方の案を提示し、推奨案を 1 つ選んでユーザーに確認します。

4. **テスト実行**: 各コミット作成前にテストを実行し、すべてが通過することを確認します。

5. **コミット履歴**: 既存のコミット履歴を維持し、新しいコミットを追加します。
